Node.js Best Practices

By Reid Chu 17 March 2021


甚麼是 node.js?

JavaScript 是前端工程的靈魂，主要目的是控制網頁的行為以及邏輯，例如產生動畫或者和使用者互動，是網路世界的共通語言。然而強大的 JavaScript 卻只出現在瀏覽器之中，在過去是個目的單純的程式語言，只當你在開啟電腦連上網路後，才會發揮作用。

node.js 在尾巴附有 js 正是 JavaScript 的縮寫，他與 JavaScript 的關係密不可分，幾乎說是同一種語言。然而他克服了只能在瀏覽器上的限制，可以在其他環境下運行，其中最重要的應用，就是可以做為伺服器編程語言。與 PHP 最大的差別是，他是 non-blocking 語言，可以同時執行多項命令的特色，可以承載高並行伺服器。

Node.js Best Practices

Node.js Best Practices 是一套使用 node.js 的餐桌禮儀，對常見的狀況作出簡潔的結論，沒有任何一個項目是三個段落以上，因此比較適合已經有寫過幾項 node.js 作業的學習者，若是剛開始的新手，可以在學習一個月之後回顧。

最大的特色就是援引各方的範例，每一個項目都會有 🔗Read More 讓你連結到實際的程式碼，例如 StackOverflow 的文章或者其他 GitHub 專案。
